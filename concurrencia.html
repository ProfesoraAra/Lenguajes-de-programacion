🔒 1. El Global Interpreter Lock (GIL)

El GIL es un cerrojo que permite que solo un hilo ejecute el bytecode de Python a la vez. Aunque parece una limitación, simplifica la gestión de memoria (específicamente el conteo de referencias) y evita condiciones de carrera en el nivel del intérprete.

INTÉRPRETE (CPython)
GIL
HILO A (Ejecutando)
HILO B (Esperando)

El GIL alterna el acceso entre hilos cada 5 milisegundos aproximadamente.

🏗️ 2. Los Tres Pilares de la Concurrencia

Threading (Hilos)

Ideal para tareas I/O Bound (Web scraping, lectura de archivos). Los hilos comparten el mismo espacio de memoria.

Multiprocessing (Procesos)

Para tareas CPU Bound (Cálculos pesados, IA). Crea instancias separadas del intérprete, evadiendo el GIL.

Asyncio (Asincronía)

Programación cooperativa basada en un Event Loop. Excelente para servidores web de alto tráfico.

Matriz de Rendimiento: I/O vs CPU

La elección de la herramienta depende de la naturaleza del bloqueo. Si el programa espera a que algo externo responda (Red/Disco), la concurrencia es la respuesta. Si el programa está saturando el procesador con cálculos, el paralelismo es necesario.

💻 Modelado Conceptual: Asyncio

La sintaxis async y await introducida en Python 3.5 permite escribir código asíncrono que se lee como si fuera síncrono, facilitando el mantenimiento.

import asyncio async def fetch_data(id): # Simula una llamada a una API await asyncio.sleep(1) return f"Datos {id} listos" async def main(): tasks = [fetch_data(i) for i in range(3)] results = await asyncio.gather(*tasks) print(results) asyncio.run(main())

Conclusión del Arquitecto

La concurrencia en Python no es una "bala de plata". Para sistemas de alto rendimiento, el desarrollador debe actuar como un arquitecto, analizando si el cuello de botella es la CPU o la latencia de red. La tendencia moderna apunta hacia Asyncio para la mayoría de las aplicaciones web, reservando Multiprocessing para el procesamiento intensivo de datos.
