Seguridad en Sistemas Concurrentes

En el Módulo 4 (Concurrencia), se destaca que el estado compartido mutable es la raíz de los problemas de sincronización. Al adoptar un enfoque funcional, reducimos la necesidad de Locks y Semáforos.

A
Eliminación de Race Conditions

Si los datos son inmutables, dos hilos pueden leerlos simultáneamente sin conflicto.

B
Prevención de Deadlocks

Menos bloqueos mutuos significa un flujo de ejecución más fluido y predecible.

🐍 Ejemplo: Transformación Pura

# Uso de map() para procesar concurrentemente from concurrent.futures import ThreadPoolExecutor data = [10, 20, 30, 40] def pure_transform(n): return n * n with ThreadPoolExecutor() as executor: results = list(executor.map(pure_transform, data))

La función pure_transform no modifica el estado global, lo que permite que el ThreadPoolExecutor la ejecute en múltiples hilos de forma segura por diseño.
