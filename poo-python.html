Python: Un Ecosistema de Objetos de Primera Clase

Para el estudiante de TI, comprender la POO en Python no es solo aprender sintaxis, es entender que todo en Python es un objeto. Desde un simple entero hasta una función o un módulo, cada entidad posee una identidad, un tipo y un valor. Este paradigma permite modelar sistemas complejos de forma modular y escalable.

2. Implementación de Clases y Objetos

La clase actúa como el plano arquitectónico. En Python, la instanciación no requiere la palabra reservada new. El método __init__ no es un constructor per se, sino un inicializador que recibe la instancia recién creada (self).

class EstudianteUNRC: # Inicializador de estado def __init__(self, nombre, matricula): self.nombre = nombre # Atributo público self.__id = matricula # Atributo "privado" def mostrar_perfil(self): return f"Estudiante: {self.nombre}" # Instanciación alumno = EstudianteUNRC("Gael", "2024-001")

Importancia de 'self':

Representa la instancia del objeto. Es obligatorio como primer parámetro en los métodos de instancia para acceder a los atributos y otros métodos del mismo objeto.

Jerarquías y Herencia Múltiple

Python soporta herencia múltiple, permitiendo que una clase derive de varias clases base. Para gestionar el orden de búsqueda de métodos, Python utiliza el algoritmo C3 Linearization, conocido como MRO (Method Resolution Order).

El Poder de super()

La función super() es esencial para llamar a métodos de la clase padre sin nombrarla explícitamente, facilitando el mantenimiento y respetando el MRO.

class Becado(EstudianteUNRC): def __init__(self, nombre, matricula, beca): # Llama al inicializador padre super().__init__(nombre, matricula) self.beca = beca

3. Encapsulamiento Idiomático

En Python, la privacidad es una convención de caballeros. No existen palabras como private. Se utilizan prefijos de guiones bajos para indicar visibilidad.

  • Público (normal): Acceso sin restricciones.
  • Protegido (_): Sugerencia de no acceso externo.
  • Privado (__): Activa Name Mangling para evitar colisiones.

4. Métodos Mágicos (Dunder)

Los métodos "Double Underscore" permiten la sobrecarga de operadores y definen comportamientos internos ante funciones integradas de Python.

__str__: Representación para usuarios.
__repr__: Representación técnica.
__len__: Respuesta a len().
__add__: Operación suma (+).
